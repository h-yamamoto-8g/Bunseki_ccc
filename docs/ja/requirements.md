**最終更新日**: 2026-04-08

## 1. アプリ全体の仕様

### 起動・終了
- スタンドアロンexeとして動作（Python環境不要）
- 起動時に `~/.bunseki/settings.json` から USERPROFILE を読み込み（未設定時はフォルダ選択ダイアログ表示）
- USERPROFILE から同期環境（SYNC_ROOT）とデータ保存先（DATA_PATH）を自動導出
- スプラッシュ画面は起動時に表示されるが、最前面固定ではない（他のアプリを操作すると背面に回る）
- デフォルト: フルスクリーン

### パス解決
- **USERPROFILE**: ユーザーのホームフォルダ（例: `C:\Users\12414`）
- **SYNC_ROOT**: `{USERPROFILE}\トクヤマグループ`
- **DATA_PATH**: `{USERPROFILE}\トクヤマグループ\環境分析課 - ドキュメント\app_data`
- ツール設定のパスでは `%USERPROFILE%`、`%SYNC_ROOT%`、`%DATA_PATH%` の変数が使用可能

### ウィンドウ
- タブ切り替え型の単一メインウィンドウ
- 最小サイズ: 1280 x 800px
- タイトルバー: "Bunseki ver.{version}"

### 計算ルール
- `bunseki.csv::test_raw_data` を使用して計算する場合は、文字列を排除して計算する。桁数は入力されているデータをそのまま使用する。

### データ表示ルール
- 数値は取得したまま表示する（pandas による float 変換を補正）
- 整数値は整数として表示（例: `0` であり `0.0` ではない）
- 小数点付きの値はそのまま表示（例: `1.234`）
- 空値・NaN は空文字として表示

- `page`: サイドバーに表示

- `dialog`: モーダルダイアログ（サイドバーに非表示）

## [page] ホーム

**目的**: 月間スケジュールと本日の未処理タスクを表示するダッシュボード。

**表示内容**:
- 月間スケジュール（QWebViewでExcelレンダリング）
- アクティブタスク一覧（自分のタスクと他人のタスクを分離）
- 新規タスクボタン

**操作**:
- 「新規タスク」ボタン → `tasks.task_ticket` へ遷移
- タスク行ダブルクリック → そのタスクの現在の `tasks.{state}` へ遷移

---

## [page] タスク

**目的**: タスクワークフロー管理の中心ページ。初期表示はタスク一覧 + 新規タスクボタン。最新100件表示。

**表示内容**:
- タスクテーブル列: 作成日時, ホルダグループ, JOB番号, ステータス, 担当者, 最終更新日, アクション
- ステータスフィルタ（全て / 進行中 / 完了 / 自分 / 担当者別）
- 「もっと読み込む」ボタン（次の100件）

### ステート遷移フロー

```
起票 → 分析対象 → 分析準備 → データ入力 → データ確認 → 回覧 → 終了
```

**遷移ルール**:
- 全ステートは他のステートからナビゲーション可能
- `task_ticket` と `analysis_targets` は「編集」ボタンで編集モード化
- 確認者に回覧後: `result_verification` と `submission` のみ編集可能

---

### [state] 起票

**目的**: 新規タスクの起票。

**表示内容**:
- ホルダグループドロップダウン（`holder_groups.json`）
- JOB番号入力 + 追加ボタン（タグ形式）
- ニュース一覧（選択したホルダグループでフィルタ）

**備考**:
- タスクID: `{作成日時}_{holder_group_code}`
- タスク名: `{作成日時}_{holder_group_name}`

---

### [state] 分析対象

**目的**: このタスクのサンプル一覧を表示・編集する。

**表示内容**:
- サンプルテーブル（設定で列の表示/非表示・順序・名称を変更可能）
- 計算列: 中央値、最大値、最小値

**データソース**:
- サンプルテーブル: `bunseki.csv`
- サンプル追加ドロップダウン: `valid_samples.json`

---

### [state] 分析準備

**目的**: 分析情報（マニュアル、ツール、チェックリスト）を提供。

**表示内容**:
- 分析前: リンク、チェックリスト
- 分析後: ログ、チェックリスト

---

### [state] データ入力

**目的**: 分析後のデータ入力と入力ツール（Excel）への連携。

**アクションバー（ボタン配置）**:
```
Lab-Aid(左端) | 分析結果の読み込み・入力ツールへ引継ぎ・データ更新・入力チェック(中央) | 一時保存(右端)
```

**各ボタンの機能**:

| ボタン | 機能 |
|--------|------|
| Lab-Aid | Lab-Aid アプリケーションを起動（設定のパスから。`.appref-ms` 等に対応） |
| 分析結果の読み込み | 後日実装予定（現在は無効状態） |
| 入力ツールへ引継ぎ | 設定済み Excel ファイルの指定シート・セルにデータを書き込み、Excel を開く |
| データ更新 | bunseki.csv の再読み込み |
| 入力チェック | `input_data` と `bunseki.csv::data_raw_data` を比較し、不一致行を赤色ハイライト |
| 一時保存 | 入力データを state_data に保存 |

**テーブル**:
- ホルダーごとにタブ分け
- 列は設定で表示/非表示・順序変更可能
- `input_data` 列のみ編集可能（黄色背景）
- Enter/Tab で次の行に自動移動

**入力ツールへ引継ぎの詳細**:
- 設定の「CSV出力設定」で選択された列のデータを書き込む
- ヘッダー行は書き込まない（データのみ）
- `.xlsm` ファイルのマクロは保持（`keep_vba=True`）
- 書き込み後、一時保存も同時に実行

**入力チェック（データ照合）の詳細**:
- 比較対象: `input_data`（入力値）と `bunseki.csv::data_raw_data`（実測データ）
- 入力が空の行はスキップ
- 不一致の行は薄い赤色背景でハイライト
- 一致行は元の色にリセット
- 結果をダイアログで件数表示

---

### [state] データ確認

**目的**: 入力データの確認と自動異常検出。

**表示内容**:
- `valid_holder_display_name` ごとのタブ
- データテーブル（設定で列の表示/非表示・順序変更可能）
- 計算列: 最上限基準値、最下限基準値、異常フラグ
- 色分けされた異常検出結果
- トレンドグラフボタン
- 確認チェックボックス

---

### [state] 回覧

**目的**: 結果と添付ファイルを確認者に回覧する。

**回覧の仕組み**:
- pywin32経由でOutlookを使用（SMTPは使わない）
- HTMLデザインの通知メール
- 異常がある場合: 警告スタイルのメール

---

### [state] 終了

**目的**: タスクの完了状態または進捗状態を表示。

**表示内容**:
- タスクが完了している場合（`current_state == "completed"`）:
  - 「タスクが完了しました」+ タスク名、ホルダグループ、JOB番号、担当者、サンプル数、起票日時、完了日時
- タスクが進捗中の場合:
  - 「タスクは進捗中です」+ タスク名、ホルダグループ、JOB番号、担当者、サンプル数、起票日時

## [page] 設定

**目的**: アプリケーション設定の編集。

### USERPROFILE セクション（タブの上部に常時表示）
- 現在の USERPROFILE パスを読み取り専用で表示
- 「変更」ボタン → フォルダ選択ダイアログ
- 導出パスのプレビュー表示:
  - 同期環境: `{USERPROFILE}\トクヤマグループ`
  - データ保存先: `{USERPROFILE}\トクヤマグループ\環境分析課 - ドキュメント\app_data`

### タブ構成

**ユーザー管理タブ**: ユーザーの追加・削除・編集

**ホーム設定タブ**: ホーム画面の設定

**タスク設定タブ**（サブタブ構成）:
- **分析項目設定**: ホルダグループごとのチェックリスト・リンク設定
- **表示列設定**: 各テーブルの列の表示/非表示・順序・名称を設定
  - 分析対象テーブル
  - データ確認テーブル
  - データ入力テーブル
  - データ入力テーブルのCSV出力設定
- **ツール設定**: Lab-Aid / 入力ツール（Excel）のパス設定

**データ設定タブ**: CSV列の表示設定

### 表示列設定の詳細
- チェックで表示/非表示を切り替え
- チェック済みの列は自動で上に詰まる（新しくチェックしたものはチェック群の末尾に追加）
- ▲▼ ボタンで表示順を変更
- 表示名を自由に変更可能
- `locked` フラグの列（例: `input_data`）はチェックを外せない

### ツール設定の詳細

**使用可能な変数**（コピーボタン付き）:
- `%USERPROFILE%` → ユーザープロファイルパス
- `%SYNC_ROOT%` → 同期環境パス
- `%DATA_PATH%` → データ保存先パス

**Lab-Aid**: アプリパスまたは URL（`.appref-ms` 対応、環境変数展開）

**入力ツール（Excel）**:
- Excel ファイルパス（環境変数展開）
- シート名（未指定時はアクティブシート）
- 開始セル（未指定時は A1）

パス入力欄の下にリアルタイムプレビュー表示:
- 環境変数を展開した後のフルパス
- ファイルが存在しない場合は赤文字で警告

## 4. 共通UI仕様

### サイドバー
- `kind=page` の画面のみ表示
- 現在のページをハイライト
- アイコンベースのナビゲーション

### タスクステートヘッダー
- ステートアイコンを上部中央に配置
- ステートはクリック可能なリンク
- 左にタスク名、右にユーザー名

### エラー処理
- 予期しないエラー: `QMessageBox.critical` でスタックトレース表示
- エラーログ: `{DATA_PATH}/bunseki/logs/app.log`
