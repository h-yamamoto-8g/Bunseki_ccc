**最終更新日**: 2026-03-01

## 1. アプリ全体の仕様

### 起動・終了
- スタンドアロンexeとして動作（Python環境不要）
- 起動時に `{app_data}/bunseki/config/settings.json` を読み込む（未設定時はセットアップダイアログ表示）
- デフォルト: フルスクリーン

### ウィンドウ
- タブ切り替え型の単一メインウィンドウ
- 最小サイズ: 1280 x 800px
- タイトルバー: "Bunseki ver.{version}"

### 計算ルール
- `bunseki.csv::test_raw_data` を使用して計算する場合は、文字列を排除して計算する。桁数は入力されているデータをそのまま使用する。

## [page] ホーム

**目的**: 月間スケジュールと本日の未処理タスクを表示するダッシュボード。

**表示内容**:
- 月間スケジュール（QWebViewでExcelレンダリング）
- アクティブタスク一覧（自分のタスクと他人のタスクを分離）
  - `tasks.{state}` 名でステータス表示
- 新規タスクボタン（`tasks.task_ticket` へ遷移）

**操作**:
- 「新規タスク」ボタン → `tasks.task_ticket` へ遷移
- タスク行ダブルクリック → そのタスクの現在の `tasks.{state}` へ遷移

**エラー**:
- アクティブタスクが見つからない場合: 赤い警告バナー + 再作成ボタン → `tasks.task_ticket`

---

## [page] タスク

**目的**: タスクワークフロー管理の中心ページ。初期表示はタスク一覧 + 新規タスクボタン。最新100件表示。

**表示内容**:
- タスクテーブル列: 作成日時, ホルダグループ, JOB番号, ステータス, 担当者, 最終更新日, アクション
- ステータスフィルタ（全て / 進行中 / 完了 / 自分 / 担当者別）
- 「もっと読み込む」ボタン（次の100件）

**操作**:
- 「新規」ボタン → `tasks.task_ticket`
- 行ダブルクリックまたはアクションボタン → そのタスクの現在のstateへ遷移
- フィルタ切り替え → 一覧をフィルタ
- 「もっと読み込む」→ 追加100件

### ステート遷移フロー

```
起票 → 分析対象 → 分析準備 → データ入力 → データ確認 → 回覧 → 終了
```

**遷移ルール**:
- 全ステートは他のステートからナビゲーション可能
- `task_ticket` と `analysis_targets` 以外は再訪問時に編集可能
- `task_ticket` と `analysis_targets` は「編集」ボタンで編集モード化。保存すると後続ステートがリセット（警告ダイアログ付き）
- 確認者に回覧後: `result_verification` と `submission` のみ編集可能。他は読み取り専用。確認者は分析者に差戻し可能。

**代理回覧**:
- 他ユーザーの未完了タスクで利用可能
- 代理ユーザーが元ユーザーの代わりにワークフローを進行可能
- 代理操作は記録され、可視的に表示される

### [state] 起票

**目的**: 新規タスクの起票。

**表示内容**:
- ホルダグループドロップダウン
- JOB番号入力 + 追加ボタン（タグ形式）
- ニュース一覧（選択したホルダグループでフィルタ）

**データソース**:
- ファイル: `holder_groups.json`
- ホルダグループ: `holder_groups.json > items[].holder_group_name`

**モード**:
- 新規: 編集可能
- 他のステートから: 読み取り専用。「編集」ボタンで編集モード化（保存で後続リセット）
- 回覧後: 完全ロック

**操作**:
- 「作成」ボタン → タスク作成、`tasks.analysis_targets` へ遷移
- 「+」ボタン → JOB番号タグ追加（管理者設定からデフォルト値）
- 作成時: 未読の重要ニュースを強制ポップアップ

**備考**:
- タスクID: `{作成日時}_{holder_group_code}`
- タスク名: `{作成日時}_{holder_group_name}`
- `holder_groups.json::holder_group_code` を保持

---

### [state] 分析対象

**目的**: このタスクのサンプル一覧を表示・編集する。

**モード**:
- **閲覧モード**（デフォルト）: 読み取り専用
- **編集モード**: 「編集」ボタンでサンプルの追加・削除が可能（編集済みマーク付き）。他のステートからの編集は後続ステートをリセット（警告付き）。回覧後はロック。

**表示内容**:
- タスク名（ヘッダー）
- サンプルテーブル: JOB番号, サンプリング日時, サンプル名, データ番号, 中央値, 最大値, 最小値

**操作**:
- 「追加」→ ドロップダウンまたは自由入力（自由入力は仮IDを作成、計算なし）
- 「削除」→ サンプルレコード削除
- 「印刷」→ 横向き印刷、列幅をページ幅に合わせる
- 「次へ」→ `tasks.analysis`
- 「戻る」→ `tasks.task_ticket`（閲覧モード）

**データソース**:
- サンプルテーブル: `bunseki.csv`
  - フィルタ: `holder_group_code` 一致 + `job_number` が選択リスト内（OR）+ 重複なし
  - 集計データ: `valid_sample_set_code` 一致、`trend_enabled` = true、`sample_sampling_date` が過去5年以内（AND）
- サンプル追加ドロップダウン: `valid_samples.json`
  - フィルタ: `items[].is_active` = true、`sort_order` 順、`display_name` 表示

**備考**:
- 選択サンプルの `bunseki.csv::valid_sample_set_code` を保持
- `bunseki.csv` にデータ番号列がない場合、すべて0をデフォルト

---

### [state] 分析準備

**目的**: 分析情報（マニュアル、ツール、チェックリスト）を提供し、ユーザーの分析完了を待つ。

**表示内容**:
- 分析前:
  - リンク: 分析基準書、関連マニュアル、作業ツール
  - チェックリスト（分析種別によっては存在しない）
- 分析後:
  - ログ: 機器、試薬（分析種別によっては存在しない）
  - チェックリスト（分析種別によっては存在しない）

**操作**:
- リンク: WebURLとファイルパスを適宜切り替え
- ログ入力（ログページと同じ。この分析種別にログが存在する場合は必須）
- チェックリスト: チェックボックス、すべてチェックで次へ進行可能。一括チェックボタンあり。回覧後はロック。
- 「次へ」→ `tasks.result_entry`
- 「戻る」→ `tasks.analysis_targets`（閲覧モード）

### [state] データ入力

**目的**: 分析後のデータ入力。（当面は最小限の実装）

**表示内容**:
- メッセージ: 「通常のデータ入力を行なってください。」

**操作**:
- 「Lab-Aid起動」ボタン → Lab-Aidアプリケーション起動
- 「入力完了」→ `tasks.result_verification`

### [state] データ確認

**目的**: 入力データの確認と自動異常検出。

**表示内容**:
- `valid_holder_display_name` ごとのタブ、各タブに:
  - データテーブル（読み取り専用）:

| カラム | 表示名 | 備考 |
|--------|--------|------|
| valid_sample_display_name | サンプル名 | valid_sample_set_code を保持 |
| valid_test_display_name | 試験項目名 | valid_test_display_name を保持 |
| test_raw_data | データ | |
| test_unit_name | 単位 | |
| - | 最上限基準値 | test_upper_limit_spec_1~4 の最小値 |
| - | 最下限基準値 | test_lower_limit_spec_1~4 の最大値 |
| test_judgment | 異常フラグ | NN: 計算結果を表示（trend_enabled=trueのみ）、それ以外はそのまま表示 |
| - | トレンド | トレンドグラフボタン |

  - 色分けされた異常検出結果
  - ハイライトされた異常フラグ
- 確認チェックボックス

**データソース**:
- ファイル: `bunseki.csv`
- 分析結果データ（OR）: `holder_group_code` 一致 + `job_number` 一致 + 重複なし
- 計算データ（AND）: `holder_group_code` + `trend_enabled` = true + `valid_sample_set_code` 一致 + `sample_sampling_date` 過去5年 + `test_raw_data` の数値のみ
- グラフデータ（AND）: 計算データと同じ + `valid_test_code` 一致

**グラフ要件**:
- 文字化けなし（日本語フォント対応）
- 単なる画像貼り付け不可 — インタラクティブなグラフ
- カーソル位置のデータを表示するツールチップ
- 上下限値（赤い破線）+ 平均±2σ（オレンジ破線）
- グラフ下部にデータテーブル（異常データはハイライト）

| カラム | 表示名 | 備考 |
|--------|--------|------|
| sample_sampling_date | 日時 | |
| test_raw_data | データ | |
| test_unit_name | 単位 | |
| test_judgment | 判定 | NN → OK、それ以外はそのまま表示 |

**操作**:
- トレンドグラフボタン → ポップアップダイアログ
- 「回覧へ」→ チェックボックス確認、`tasks.submission` へ遷移

### [state] 回覧

**目的**: 結果と添付ファイルを確認者に回覧する。

**表示内容**:
- 添付ファイル欄（ドラッグ＆ドロップまたはダイアログ、ファイルは `{app_data}` にタスクIDをキーとして保存）
- 回覧フロー（分析者 → 確認者、確認者の人数は可変）
- コメント欄

**回覧の仕組み**:
- SMTPは使わない — pywin32経由でOutlookを使用（メール作成・表示、ユーザーが送信）
- HTMLデザインの通知メール（プロフェッショナルな外観）
- メール作成時にステータス変更
- 異常がある場合: 警告スタイルのメールと異常テーブル（データ確認テーブルからフィルタ）
- 平均±2σを超える項目: `[page] data` 表示用の異常リストに記録。`bunseki.csv` から保存: `sample_request_number`, `sample_job_number`, `valid_sample_set_code`, `valid_holder_set_code`, `valid_test_set_code`

**操作**:
- 作成者の役割 → 「送信」ボタン（確認者に通知、作成者の作業完了）
- 確認者の役割 → 「転送」（次の確認者を選択）または「完了」→ `tasks.completed`

## 4. 共通UI仕様

### サイドバー
- `kind=page` の画面のみ表示（stateやdialogは非表示）
- 現在のページをハイライト
- アイコンベースのナビゲーション

### タスクステートヘッダー（`tasks.{state}`）
- ステートアイコンを上部中央に配置、左右にナビゲーションボタン
- アクティブなステートはその表示名を表示
- ステートはクリック可能なリンク
- 左にタスク名、右にユーザー名
- タスク以外のページ: グレーアウトされたアイコン
- コンテンツは垂直スクロール可能
- 下部中央に「次へ」と「戻る」ボタン

### ステータスバー（メインウィンドウ下部）
- 現在の処理状態を表示（例: 「読み込み中...」「準備完了」）

### エラーダイアログ
- 予期しないエラー: `QMessageBox.critical` でスタックトレース表示
- エラーログ: `{app_data}/bunseki/logs/app.log`

### エラー処理
- SharePointフォルダが見つからない: 赤い警告バナー + 設定ボタン → `setup` ダイアログ → データ再読み込み
- エラー: 赤い警告バナーにエラー詳細 + 「管理者に連絡」メッセージ + エラーログダウンロードボタン
